Bonding Curve AMM

Constant-product formula (pump.fun style)

// Constant-product AMM: k = VP × VT fn calculate_buy(vp: u64, vt: u64, quote_in: u64) -> u64 { let k = (vp as u128) * (vt as u128); let new_vp = vp + quote_in; let new_vt = k / new_vp; let tokens_out = vt - new_vt; // tokens to buyer tokens_out } // Virtual reserves provide initial liquidity without real tokens // VP = 843,800 PIGEON (configurable per quote asset) // VT = 1,073,000,000 tokens // Real reserves track actual deposited amounts // Graduation triggers when real_quote >= threshold

Price Discovery: Early buyers get more tokens per unit.
As demand grows, price increases along the curve.
Fair launch — no presale, no team allocation.

Transfer Hook

Token-2022 extension — post-graduation fee engine

// Token-2022 calls hook on every transfer_checked pub fn execute(ctx, amount) { // Pre-graduation: passive (no-op) if !config.active { return Ok(()); } // Post-graduation: accrue 0.25% fee let fee = amount * 25 / 10_000; vault.total_accrued += fee; // Non-reverting: errors → Ok(()) + log // Never blocks user transfers } // Sweep: cranker calls periodically pub fn sweep_fee_accrual(ctx, route_data) { // On-chain Jupiter swap → PIGEON // burn_checked → reduce supply // Permissionless, single-mint }

Key Design Decisions

  • Non-reverting: Never blocks transfers
  • Parametric mint: Not hardcoded
  • Per-mint scoped: FeeAccrualVault per token
  • Jupiter CPI: On-chain swap + burn
  • Permissionless sweep: Anyone can crank
